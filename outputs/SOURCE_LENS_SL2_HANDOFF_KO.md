## 구현 결과

### 코드 파츠 번역

- AST 근거로 코드 단위를 선언, 명령, 가정·분기, 반복, 응답·반환으로 나눈다.
- 리소스, 설정, 데이터 참조를 별도 파츠로 분류한다.
- 각 파츠는 `파일 + AST 노드 종류 + 심볼 + 범위 + 주변 구조 fingerprint`로 만든 안정 앵커를 가진다.
- 자연어 설명은 결정적 한국어 템플릿으로 만든다. 설명과 실제 파일·줄 범위 근거를 항상 함께 보여준다.
- 코드 브라우저의 모듈 상세와 캔버스 모듈 노드에서 같은 파츠 계약을 재사용한다.
- `editable` 자리는 스키마에 포함했지만, SL-3에서 명시 등록된 속성을 제외하면 모두 읽기 전용이다.

### 모듈 실체화

- 코드 브라우저의 `캔버스에 올리기`와 모듈 드래그·드롭을 모두 구현했다.
- 두 경로 모두 캔버스를 즉시 바꾸지 않고 기존 Reconciliation Proposal을 만든다.
- 사용자는 미리보기 후 승인해야 하며 자동 대량 실체화는 계속 금지한다.
- candidate, 근거 미확인, 작동 미확인 대상은 dimmed로 표시한다. dimmed는 Reality Level을 높이지 않는다.

### AI 설명 파일럿 경계

- 공급자 중립 서버 어댑터를 구현했지만 기본값은 비활성이다.
- 소유자와 Workflow Canvas 자기 시스템 지도·자기 저장소 조합에서만 요청할 수 있다.
- 브라우저는 AI 키를 받지 않는다. 서버에는 소스 본문이 아니라 제한된 AST 메타데이터만 전달한다.
- AI 문장은 `AI 설명` artifact로 표시되고 결정적 설명·근거와 나란히 보인다.
- AI는 관계, 권한, Reality Level, 캔버스 실체를 만들 수 없다.
- 환경변수 이름만 준비했다: `SOURCE_LENS_AI_ENABLED`, `SOURCE_LENS_AI_PROVIDER`, `SOURCE_LENS_AI_MODEL`, `SOURCE_LENS_AI_API_KEY`.
- 키·모델·예산·전송 정책은 사용자 별도 승인 전 설정하지 않는다.

## 성능 경계

- 코드 단위 전체 manifest는 서버 전용으로 유지한다.
- 브라우저는 선택한 모듈의 파츠만 `/api/source-twin?mode=code-parts`로 지연 로드한다.
- 브라우저 엔트리가 서버 전용 manifest를 import하지 못하게 회귀 테스트를 추가했다.
- 현재 서버 생성물은 약 3.12 MB이며 대형 저장소 샤딩·캐시는 `PERF-004`에 계속 추적한다.
