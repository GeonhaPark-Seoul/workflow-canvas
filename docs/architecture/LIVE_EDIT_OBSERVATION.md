# 라이브 수정 관측 메모

> 이 문서는 특정 시스템 지도 레이어에 속하지 않는 공통 관측 기능의 설계 메모다.
> 아직 구현 계약이 아니며, 첫 사용 화면을 검토할 때 다시 범위와 UI를 확정한다.

| 항목 | 값 |
|---|---|
| 문서 버전 | 0.1.0 |
| 작성일 | 2026-07-24 |
| 상태 | 레이어 독립 후보 기능 |

## 1. 목적

사람이나 AI 에이전트가 지금 어느 코드 영역을 수정하고 있는지 보여준다. 작업 주체를
지도 노드로 만드는 기능이 아니다. 수정 대상 Asset의 상태·타임라인·배지로 관측한다.

이 기능은 다음 두 기반이 필요하다.

1. 워크숍 카드가 자유 텍스트가 아니라 실제 `assetId`를 참조한다.
2. 같은 Asset을 표현하는 여러 화면의 노드는 공통 관측 상태를 함께 읽는다.

## 2. 관측 신호

| 신호 | 의미 | 근거 수준 | 보관 방식 |
|---|---|---|---|
| 작업 주체의 자기 보고 | “지금 파일 X를 편집 중”이라는 활동 신고 | `declared` | 짧은 TTL을 가진 일시 상태 |
| 로컬 커넥터의 독립 관측 | 실제 작업공간의 변경 파일 확인 | `discovered` 또는 `observed` | 관측 시각과 근거를 기록 |
| 커밋 확인 | 변경이 특정 커밋으로 남았음을 확인 | `discovered` 또는 `observed` | Asset 타임라인의 지속 기록 |

자기 보고를 독립 관측으로 승격하지 않는다. 두 신호가 다르면 그 불일치 자체를
사용자에게 보여줄 수 있어야 한다.

## 3. 표현 원칙

- 작업 주체, 편집 세션, 커밋은 이 기능만을 위해 새 노드로 만들지 않는다.
- 파일 경로는 코드 Asset 계층에 해석하고, 화면에는 필요한 집계 수준의 파츠나 배지로
  표시한다.
- 일시적인 “편집 중” 상태와 append-only 감사 기록은 저장 성격이 다르므로 분리한다.
- 볼 권한이 없는 Asset의 존재가 워크숍 카드나 관측 배지를 통해 새어 나오지 않도록
  동일한 redaction 경계를 적용한다.
- 이 기능은 코드 구조, 개발 작업공간, 운영 관측 등 여러 화면에서 같은 `assetId`를
  통해 재사용할 수 있다.

## 4. 구현 전 확인할 것

- 자기 보고의 TTL과 갱신 주기
- 로컬 커넥터에 허용할 최소 읽기 범위
- 새 파일을 기존 코드 Asset에 연결하기 전의 임시 표시 방식
- Asset 삭제·병합 시 카드 참조의 보존 정책
- 자기 보고와 독립 관측이 다를 때의 UI
